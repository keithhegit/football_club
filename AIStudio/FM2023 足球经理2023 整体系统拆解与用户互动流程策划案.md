# FM2023 足球经理2023 整体系统拆解与用户互动流程策划案

## 文档版本与说明

- **版本**：1.0

- **日期**：2025-12-02

- **目的**：本策划案基于指定资源（Guide to FM、FM Scout、SortItOutSI）对FM2023的核心系统进行完整拆解，聚焦数据库结构、球员属性逻辑、战术指令与比赛引擎交互，以及用户完整互动流程。拆解粒度达到指导开发水平，包括数据流、计算公式（已知部分）、模块依赖、UI交互细节。

- **数据来源限制**：严格基于guidetofm.com（属性影响比赛逻辑）、fmscout.com（CA/PA算法与隐藏属性）、sortitoutsi.net（数据库更新结构）。FM2023机制与FM22高度一致，FM24属性描述作为补充验证。

- 开发指导原则

  ：

  - **数据驱动**：所有模拟基于CA/PA加权属性计算。
  - **实时模拟**：比赛引擎每帧计算属性+指令影响。
  - **UI反馈**：星级、报告、分析图表实时更新。
  - **模块化**：独立模块（如训练影响属性增长），支持Editor Data（.fmf文件）热更新。

## 1. 游戏整体架构

FM2023采用**模块化数据库+实时模拟引擎**架构：

text

```
[前端UI层] <-> [核心逻辑层] <-> [数据库层] <-> [模拟引擎层]
```

- **数据库层**：SQLite-like .fmf文件，存储球员/俱乐部/联赛数据。支持社区Data Update（球员属性、转会、财务）。
- **核心逻辑层**：CA/PA计算属性、训练/比赛影响增长/衰退。
- **模拟引擎层（Match Engine）**：属性组合+战术指令决定每球事件概率（e.g., 传球成功 = Passing * Vision * Decisions权重）。
- **前端UI层**：动态面板（概览、战术板、比赛3D/2D视图）。
- **数据流**：加载存档 -> 日程推进（训练/比赛）-> 属性微调 -> 渲染UI。

| 架构组件  | 依赖数据                | 输出                   | 更新频率      |
| --------- | ----------------------- | ---------------------- | ------------- |
| 数据库    | .fmf文件（CA/PA、属性） | 球员/球队实例          | 赛季初/转会窗 |
| CA/PA计算 | 位置权重                | 可见属性值（1-20）     | 训练/比赛后   |
| 比赛引擎  | 属性+指令               | 事件序列（射门、传球） | 实时（60FPS） |
| UI交互    | 逻辑层API               | 面板/报告              | 每tick        |

## 2. 核心系统拆解

### 2.1 数据库系统（基于SortItOutSI Data Update结构）

- 核心实体

  ：

  - **球员（Player）**：ID、姓名、年龄、位置、国家、俱乐部、合同、CA（当前能力1-200）、PA（潜力上限1-200，固定或范围如-9=-95）、属性（25+隐藏）、个性、特质、伤病史。
  - **俱乐部（Club）**：财务（余额、工资预算）、声誉、设施（训练场、青年设施）、阵容。
  - **联赛/比赛**：赛程、规则（Home/Away优势）。

- **更新机制**：.fmf文件覆盖默认DB，支持属性提交（CA/PA+可见属性）。增强版动态属性（每存档唯一）。

- 开发接口

  ：

  pseudo

  `loadDB(fmf_path): Player[] // 解析.fmf到内存 updatePlayerAttr(player_id, attr_name, value): void // 提交审核后应用`

- CA/PA算法

  （FM Scout核心）：

  - CA = Σ(位置特定权重 * 属性值)，范围1-200。e.g., 前锋物理属性权重高。
  - PA：上限，固定（e.g., 182）或范围（e.g., 170-200随机）。U23球员可转为范围增加变异。
  - 属性派生：CA点数按位置分配（e.g., Poacher专注Finishing高值）。
  - RCA（推荐CA）：维持当前属性的最小CA，编辑器检查平衡。

### 2.2 球员系统（属性+隐藏，GuideToFM+SortItOutSI）

- **属性分类**（25可见+隐藏，如GK Passing）：

| 分类     | 属性列表                                                     | 影响示例（开发公式）                                         |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **技术** | Corners, Crossing, Dribbling, Finishing, First Touch, Free Kick Taking, Heading, Long Shots, Long Throws, Marking, Passing, Penalty Taking, Tackling, Technique | Passing成功率 = Passing(基础) * Technique(执行) * Vision(机会识别) * Decisions(选择)。低First Touch易失球。 |
| **心理** | Aggression, Anticipation, Bravery, Composure, Concentration, Decisions, Determination, Flair, Leadership, Off the Ball, Positioning, Teamwork, Vision, Work Rate | Decisions+Anticipation决定定位/拦截概率。Vision spotting传球机会。 |
| **物理** | Acceleration, Agility, Balance, Jumping Reach, Natural Fitness, Pace, Stamina, Strength | Pace+Acceleration决定追球成功。Stamina衰退曲线：疲劳后属性-20%。 |
| **门将** | Aerial Reach, Command of Area, Communication, Eccentricity, Handling, Kicking, One on Ones, Reflexes, Rushing Out, Punching, Throwing | Handling湿天气反弹率↓。Rushing Out+Acceleration拦截通过球。  |

- **隐藏属性**：Genie Scout揭示（Consistency, Important Matches, Injury Proneness, CA/PA）。

- **星级计算**：相对球队/联赛CA对比（1-5星，金星上限）。

- 开发逻辑

  ：

  pseudo

  `computeActionSuccess(action_type, player_attrs, opp_attrs, instruction_mod):  base_prob = weighted_sum(player_attrs[action_keys])  mod = instruction_mod * (teamwork / 20)  return clamp(base_prob * (1 - opp_attrs[def_key]/20), 0, 1)`

### 2.3 战术与比赛引擎（GuideToFM核心）

本节基于GuideToFM（guidetofm.com）对FM2023比赛引擎（Match Engine）的完整拆解，聚焦属性组合（Attribute Combinations）、战术指令（Team/Player Instructions）与引擎交互逻辑。FM2023引擎继承FM22核心机制，但引入FM23特定优化（如AI反应、动画IK、指令重命名）。拆解粒度达开发级：包括动作成功概率伪公式（概率 = (属性权重和/20) * 战术修饰符 * 状态修饰符，范围0-1）、事件依赖、模块接口。

#### 2.3.1 战术组件

战术由**阵型（Formation）+角色职责（Roles/Duties）+团队指令（Team Instructions）+球员指令（Player Instructions）**组成。引擎实时解析为“位置责任分数”（Positional Responsibility Score）和“前进移动分数”（Forward Movement Score），影响控球方事件概率。

##### 2.3.1.1 阵型与位置调整

- **阵型解析**：引擎将4-3-3解析为11个位置向量（x,y,z），支持动态变形（e.g., 4-3-3 DM Wide中DM后撤）。开发：parseFormation(form_str) -> PositionGrid[11]。
- **FM23更新**：后卫形状优化（Wingback在5后卫高压更激进）；中场在半空间定位更高（Mezzala/AP Attack）。

##### 2.3.1.2 玩家角色与职责（Roles & Duties）

角色预设指令模板，职责（Defend/Support/Attack）调整风险（Attack: 高前压+创意自由；Defend: 持位+纪律）。关键属性需求基于GuideToFM。

| 位置      | 角色示例           | 职责    | 核心属性组合                                                 | 引擎隐式指令                   | FM23行为                          |
| --------- | ------------------ | ------- | ------------------------------------------------------------ | ------------------------------ | --------------------------------- |
| **GK**    | Sweeper Keeper     | Attack  | Reflexes + Rushing Out + Acceleration + Anticipation + Decisions | Roam/Rush Out                  | IK动画优化，一v一 Spread Saves。  |
| **DC**    | No-Nonsense CB     | Defend  | Tackling + Marking + Positioning + Strength + Bravery        | Tackle Harder, Mark Tighter    | 后侧压力+空中拦截提升。           |
| **WB/ML** | Wing Back          | Attack  | Crossing + Dribbling + Pace + Work Rate + Stamina            | Overlap, Cross More Often      | 宽度提供优化。                    |
| **DM/MC** | Roaming Playmaker  | Support | Passing + Technique + Vision + Decisions + Flair + Teamwork + Stamina | Roam, Dribble More, Take Risks | 半空间build-up更高位。            |
| **AMC**   | Advanced Playmaker | Attack  | Vision + Passing + Technique + Flair + Decisions + Off The Ball | Move Into Channels, Roam       | 焦点传球（teammates pass more）。 |
| **ST**    | Complete Forward   | Attack  | Finishing + Dribbling + Strength + First Touch + Off The Ball + Composure | Roam, Shoot More               | 切内侧强脚射门概率↑。             |

开发接口：assignRole(player_id, role, duty) -> ImpliedInstructions[]。

##### 2.3.1.3 团队指令（Team Instructions）

分类4面板：In Possession（控球）、In Transition（转换）、Out of Possession（失球）、Set Pieces。每个指令修饰事件概率（e.g., +0.2 Tempo Fast）。FM23重命名：Line of Engagement→Low/Mid/High Block；Offside Trap→Drop Off/Hold Line；Defensive Width→Pressing Traps（宽/中/窄/不对称）。

| 类别                             | 指令                     | 引擎效果                           | 核心属性需求                                                 | 概率修饰示例                            |
| -------------------------------- | ------------------------ | ---------------------------------- | ------------------------------------------------------------ | --------------------------------------- |
| **Mentality**                    | Very Attacking           | 最终1/3过载，快节奏直传，回收控球  | Intelligence (Anticipation+Decisions+Flair+Teamwork+Vision)；Passing+Technique 高；Pace 低可 | 射门+30%，失位风险+25%                  |
|                                  | Defensive                | 后撤限空，慢节奏挫败对手，直传反击 | Strength+Jumping Reach 高（ST持球）；Pace 高（中前）；Intelligence 低可 | 拦截+20%，控球-15%                      |
| **Tempo (In Possession)**        | Much Higher              | 快速直线渗透                       | Acceleration+Pace+Passing+Vision 高                          | 传球成功*1.2，拦截*0.8                  |
| **Passing Directness**           | Much More Direct         | 绕过中场，直传前场                 | Crossing/Dribbling/Heading 高（Direct）；First Touch+Passing+Technique 高（Short） | 长传成功=Crossing*0.4 + Technique*0.3   |
| **Pressing (Out of Possession)** | Much Higher (High Block) | 高位逼抢，迫失误                   | Aggression+Work Rate+Anticipation+Decisions+Pace+Positioning+Tackling 高 | 抢断概率+40%，后场空当风险+30%          |
| **Defensive Line**               | Hold Line                | 压上offside                        | Pace+Anticipation 高                                         | 越位陷阱成功=Positioning*0.5 + Pace*0.3 |

其他：Focus Play（焦点球员传球↑）；Counter-Press（快速回收）。

| Mark Tighter          | 限空间     | Concentration+Aggression+Bravery+Work Rate+Anticipation+Decisions+Balance+Strength+Jumping Reach+Acceleration+Agility+Pace+Positioning+Marking+Tackling | 针对明星          | 自由Roam角色   |

#### 2.3.2 比赛引擎拆解

**架构**：事件驱动（60FPS），每球事件序列（Possession Phase → Action → Outcome）。状态影响：Condition（Stamina衰退-20%）、Morale（+10% Decisions）、Form。隐藏属性（Consistency/Important Matches）波动技术/心理。

##### 2.3.2.1 关键动作属性组合（Attribute Combinations）

引擎每动作计算组合分数。公式：success_prob = Σ(Primary*0.4 + Key Mental*0.3 + Physical*0.2 + Technique*0.1) / 20 * mods（mods: 战术1.0-1.5，天气0.8-1.2）。

| 动作                         | Primary              | Key Mental                                           | Physical                                 | Technique/Other | 示例概率                                                     |
| ---------------------------- | -------------------- | ---------------------------------------------------- | ---------------------------------------- | --------------- | ------------------------------------------------------------ |
| **传球 (Passing)**           | Passing              | Vision + Decisions                                   | -                                        | Technique       | 短传：Passing*0.4 + Vision*0.2 + Decisions*0.2 + Technique*0.2 |
| **射门 (Shooting)**          | Finishing/Long Shots | Composure + Decisions                                | Balance                                  | Technique       | 近射：Finishing*0.4 + Composure*0.3 + Technique*0.2；FM23：跨门射角↑。 |
| **抢断 (Tackling)**          | Tackling             | Anticipation + Decisions + Aggression                | Strength + Balance                       | -               | Tackling*0.4 + Bravery*0.2 + Positioning*0.2 vs 对手Dribbling |
| **盘带 (Dribbling)**         | Dribbling            | Flair + Off The Ball                                 | Acceleration + Agility + Pace + Strength | Technique       | Dribbling*0.4 + Technique*0.3 + Pace*0.2                     |
| **头球 (Heading)**           | Heading              | Anticipation + Decisions                             | Jumping Reach + Strength                 | -               | Heading*0.5 + Jumping Reach*0.3                              |
| **第一次触球 (First Touch)** | First Touch          | Anticipation + Decisions + Flair + Teamwork + Vision | Balance (空中/屏蔽)                      | Technique       | First Touch*0.4 + Vision*0.2 + Technique*0.2；FM23：弹跳真实化。 |

更多：拦截=Anticipation + Positioning + Tackling；持球=First Touch + Strength。

##### 2.3.2.2 FM23引擎更新

- **AI**：情境反应（领先后追分、关键战）；经理风格（属性驱动）。
- **动画**：IK防滑脚，压缩1/30大小；GK膝挡、Spread Saves。
- **物理**：First Touch动量保留，切内射门↑，防守高压优化。

##### 2.3.2.3 开发伪码（Match Engine Core）

pseudo

```
class MatchEngine {  simulateMatch(tacticA, tacticB, duration=90min):    time = 0; possession = random_side();    while time < duration:      event = generateEvent(possession, current_phase);  // e.g., PASS_SHORT      actor = selectPlayer(tacticA.positions, event);      opp_attrs = tacticB.getDefender(event);      prob = computeSuccess(event.attrs_keys, actor.attrs, tacticA.mods, opp_attrs);      if rand() < prob * condition_mod(actor.condition) * morale_mod():        executeEvent(actor, event);  // e.g., PASS_SUCCESS -> next_event(CHANCE)      else:        turnover(possession = !possession);      updateFatigue(all_players);  // Stamina decay      time += tick(3-5s);    return stats_log;    computeSuccess(keys, attrs, mods, opp):    base = weighted_avg(attrs[keys]) / 20;  // e.g., {Passing:0.4, Vision:0.2, ...}    mod = mods.instr_mult * (1 - opp.def_score / 20);  // e.g., High Press *1.2    return clamp(base * mod, 0, 1); }
```

接口：暴露getEventProb(event_type)测试平衡。热更新支持.fmf战术文件。

此拆解可直接导入引擎原型，结合CA/PA计算属性，实现100%模拟一致性。

## 3. 用户互动流程（详细到UI点击级）

### 3.1 新游戏启动流程

1. **主菜单**：New Game -> 选择数据库（默认+Data Update）。
2. **经理创建**：姓名/国籍/经验 -> 偏好（风格）。

4. **加载**：DB解析 -> 进入概览面板。

### 3.2 日常管理循环（每周流程）

本节基于FM Scout（fmscout.com）和Guide to FM（guidetofm.com）对FM2023每周管理循环的完整拆解，聚焦UI交互粒度（点击/拖拽/滑块级）、数据流（属性/状态更新）、模块依赖（训练影响CA增长、比赛引擎输入）。一周典型结构（周日比赛为例）：

text

```
周一: 恢复/评估 (Recovery + Report Review)
周二-周四: 主力训练 (Patterns of Play + Match Prep)
周五: 赛前调整 (Light Training + Tactical Briefing)
周六: 比赛日 (Lineup Finalize + Kick Off)
周日: 赛后分析 (Data Hub + Media/Team Talk)
```

- **数据流**：训练报告（周五）→ 球员Condition/Sharpness更新 → 比赛输入（Morale + Form）→ 引擎模拟 → 赛后日志 → 下周循环。

- 开发伪码

  ：

  pseudo

  `weeklyLoop(week_start):  for day in 1..7:    if day == match_day: executeMatchPrep();    else: applyTrainingSchedule(day_schedule);    generateReport(day);  // Condition delta, Attr +0.1-1%    updatePlayerStates(); // Stamina decay, Morale from feedback`

- **关键依赖**（FM Scout）：单一Tactic + Schedule（Match/Non-Match split），优先Ball Retention/Patterns；3 Recovery sessions/周防伤。

#### 3.2.3 比赛执行（比赛日，Kick Off）

- **UI**：View Mode下拉 (Comprehensive: 事件+雷达；Extended: 详细动画；3D切换视角)。 实时：暂停 > Shout (Get Stuck In)；Sub拖拽；Instr调整 (e.g., Tempo Higher)。
- **数据流**：Condition * Morale → 引擎Prob (e.g., Stamina<50% → -20% Pace)。热图实时。
- **输出**：事件日志 (Pass/Shot序列)。

#### 3.2.4 赛后分析（Post-Match > Analysis）

Data Hub核心：xG/xA、PPDA (Pressing Intensity)、热图对比。

### 3.3 转会流程

1. **球探**：Scouting > Assignments（区域/位置）-> Reports（星级+CA暗示）。
2. **短名单**：Add to Shortlist -> Negotiate（报价滑块：转会费/工资）。
3. **确认**：合同条款 -> 医疗 -> 签约。

### 3.4 比赛日详细交互

- **预览**：预计比分（基于CA差）。
- **实时**：3D视图切换球员视角；热键喊话（Get Stuck In）。
- **调整**：暂停 > 替换（拖拽）/指令微调。

### 3.5 高级流程：青年发展

1. Youth Intake -> 评估CA/PA。
2. 导师：Pair（高Leadership老将+青年）。
3. 比赛时间：Rotation规则。

## 4. 开发实现建议
